# Case Template

Use this for one representative colour output, eval row, pulse, or bounded
slice that shows a Huey anchor, drift, correction, or edge case clearly.

## Metadata

| Field | Value |
| --- | --- |
| Code | `NNN_CASE` |
| Category | `case` |
| Status | `representative`, `anchor`, `counted_seam`, `excluded_noise`, or `archived` |
| Last evidence | `YYYY-MM-DD` |
| Owns | one sentence naming the case this doc stands in for |

## Headline Shape

- `Case: Name`
- or `Representative Case: Name`
- or `Bounded Case: Name`

## Section Order

1. metadata table
2. `Case`
3. `Why It Matters`
4. `Follow-Up`

## Required Case Moves

- name the exact output id, row, range, pulse, or bounded slice
- state the judged object and verdict unit explicitly
- keep the observed shape concrete
- state the verdict plainly
- name the family lane and replacement pair when the case is an eval row
- explain why this case is representative rather than merely memorable

## Default Case Table

| Field | Value |
| --- | --- |
| Artefact | output id, row id, pulse id, or bounded range |
| Surface | family rank, one-up replacement, bounded pulse, or proof surface |
| Family lane | active family or comparison lane |
| Judged object | row, pulse, split pulse, or correction |
| Observed | compact observed shape |
| Verdict | `pass`, `fail`, `anchor`, `counted_seam`, `excluded_noise`, or `retire` |

## Default Why-It-Matters Table

| Signal | Read |
| --- | --- |
| representative shape | one sentence naming the method consequence |

## Case Questions To Answer

- what exact output, artefact, or bounded slice does this stand for?
- what lens judged it?
- what happened in the case?
- why is this one representative of the wider lane?
- what should happen next because of it?

## Style Rules

- lead with the metadata table
- keep the case table literal and compact
- prefer one representative case over mini-inventories
- include hex, swatch, or replacement names only when they matter to the read
- keep the interpretation in one small table or one short paragraph
- keep follow-up immediate and operational
